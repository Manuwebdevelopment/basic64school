# Level 0 — Welcome & Basics

## Overview

Welcome to **Basic64School**! Level 0 is your friendly, zero-to-hero introduction to Commodore 64 BASIC programming. You don't need any prior knowledge — just curiosity and a willingness to play around!

### What Is the C64?

The Commodore 64 is one of the most iconic computers ever made. Released in **1982**, it sold over **17 million copies** and was the best-selling computer of its time. It had:

- **38,914 bytes of RAM** — that's where all your code and data lives
- **A 40-column × 25-line text screen** — your canvas for printing things
- **16 colors** — for border and background
- **A built-in BASIC interpreter** — meaning you can type commands and instantly see results, no compilers or fancy setup needed

### Level 0: What You Will Learn

In this level, you'll master the fundamental tools that let the C64 talk to you. Every C64 program, no matter how complex, starts with these basics:

| Lesson | Title | What You'll Learn |
|------|-----|-----|
| 0 | `01-welcome` | **Welcome** — Orientation, meeting the C64, first PRINT command, CHR$(147) clearing, and first POKE at the border color |
| 1 | `02-print-chr` | **PRINT & CHR$** — PRINT command deep dive, CHR$() for character codes, special PETSCII symbols, screen clearing, and color controls |
| 2 | `03-mem-poke` | **POKE & Memory** — Reading and writing memory, screen memory at address 1024, placing characters directly into memory |
| 3 | `04-petscii-art` | **PETSCII Art & Drawing** — Creating pictures from character codes, color graphics mode (GROK), and making your first pixel art |
| Cap | `cap-first-star` | **Capstone: First Star** — Pick coordinates, clear the screen, and place a star on the screen with POKE — your first complete program! |

### How to Use This Level

Each lesson builds on the last. Start with Lesson 0 (Welcome) and work your way through in order. Don't skip around! The C64 BASIC concepts are like building blocks — each new one rests on the ones before it.

**Every lesson includes:**

- **Theory** — Explains the concept with kid-friendly analogies
- **Key Takeaway** — The one thing to remember most from this lesson
- **C64 BASIC Examples** — Ready-to-type code you can run immediately
- **Exercise** — Step-by-step tasks to practice what you learned
- **Practice Challenge** — A fun mini-project to apply everything
- **Quiz** — 3-4 multiple-choice questions to test your understanding
- **Summary** — Bullet-point recap of key concepts
- **What's Next** — A teaser for the following lesson

**You'll type real C64 BASIC code on every lesson.** All code examples are:

1. Accurate for the actual Commodore 64
2. Runnable in any C64 emulator (like VICE)
3. Runnable in the Basic64School browser simulator
4. Designed to give you instant, visible results

### Quick C64 Facts

| Fact | Detail |
|--|----|
| **Screen size** | 40 characters wide × 25 lines tall |
| **Total screen cells** | 1,000 character positions |
| **RAM** | 38,914 bytes available to BASIC |
| **Colors** | 16-color palette (codes 0-15) |
| **Border color** | Memory address 53281 |
| **Background color** | Memory address 53280 |
| **Screen memory** | Starts at address 1024 |
| **PETSCII** | The character set the C64 uses (not ASCII!) |
